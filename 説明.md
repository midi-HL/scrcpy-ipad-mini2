\# Scrcpy-iOS 项目完整说明



\## 1. 项目是什么？



这个项目本质上是：



> \*\*把电脑上的 scrcpy 客户端移植到越狱 iPad 上，让 iPad mini 2 可以像电脑一样远程控制 Android 手机。\*\*



官方 scrcpy 的工作方式：



```

电脑

&#x20;|

&#x20;| ADB

&#x20;|

Android 手机

```



电脑负责：



\* 接收 Android 屏幕画面

\* 解码视频

\* 显示窗口

\* 发送鼠标键盘操作



而这个项目做的是：



```

iPad mini 2

(iOS 12.5.7 越狱)



&#x20;       |

&#x20;       | WiFi ADB

&#x20;       |

&#x20;       ↓



Android 手机



&#x20;       |

&#x20;       |

scrcpy-server

```



也就是说：



\*\*把原本电脑上的 Client 部分搬到 iPad。\*\*



Android 端基本不改变，继续运行官方 scrcpy-server。



\---



\# 2. 最终实现后的使用体验



安装 `.deb` 后：



iPad 上会有一个 App：



```

scrcpy-iOS

```



打开以后：



1\. 输入 Android 手机 IP



例如：



```

192.168.1.100:5555

```



2\. 点击连接。



3\. iPad 显示 Android 手机实时画面。



4\. 可以直接：



\* 点击 Android 屏幕

\* 滑动

\* 长按

\* 操作 App

\* 玩一些简单游戏

\* 查看手机内容



效果类似：



```

电脑运行 scrcpy

```



只是：



```

电脑 → iPad

```

\# 3. 核心功能



\## 3.1 WiFi ADB 连接



这是整个项目的入口。



Android 开启：



```

ADB TCP/IP

```



例如：



```

192.168.1.100:5555

```



iPad 通过 ADB 连接：



```

adb connect



↓



adb devices



↓



建立控制通道

```



之后：



iPad 可以：



\* 启动 scrcpy-server

\* 接收视频

\* 发送控制命令



\---



\# 3.2 Android Scrcpy Server



Android 端运行：



```

scrcpy-server.jar

```



它负责：



\## 屏幕捕获



Android：



```

Surface



↓



MediaCodec



↓



H264 Encoder

```



产生：



```

H264 视频流

```



\---



\## 接收控制



例如：



iPad 点击：



```

x=500

y=800

```



转换：



```

Control Message



↓



ADB



↓



Android Input

```



手机执行点击。



\---



\# 3.3 视频传输系统（核心）



这是项目最重要部分。



完整流程：



```

Android屏幕



↓



H264编码



↓



ADB Socket



↓



iPad接收



↓



Demuxer



↓



VideoToolbox



↓



CVPixelBuffer



↓



OpenGL ES



↓



屏幕显示

```



\---



\## 为什么不用 FFmpeg？



电脑版本：



```

H264



↓



FFmpeg



↓



Frame

```



但是 iPad mini 2：



A7 芯片。



性能有限。



所以使用：



```

VideoToolbox

```



这是 Apple 自带硬件解码框架。



优势：



\* 使用 GPU/硬件解码

\* 更省电

\* 延迟更低

\* CPU 占用低

\# 11. 为什么需要越狱？



因为普通 iOS：



限制：



\* 后台运行

\* ADB工具

\* Socket权限

\* 文件访问

\* 系统Framework调用



越狱后：



可以：



\* 安装 deb

\* 使用 Theos

\* 运行自定义程序

\* 调用底层接口



\---



\# 12. 为什么选择 iPad mini 2？



虽然性能低，但：



优势：



\* ARM64

\* 支持 OpenGL ES

\* 支持 iOS 12

\* 可以越狱

\* 屏幕适合显示手机画面



限制：



\* 1GB RAM

\* A7 CPU



所以：



设计目标：



```

720p

30FPS

低延迟

```



而不是：



4K。
